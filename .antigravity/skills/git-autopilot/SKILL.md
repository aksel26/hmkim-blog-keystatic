# Git Autopilot Skill

## Description
변경된 코드의 의미를 분석하여 논리적 단위(Atomic Commits)로 분류하고, 
각 단위별로 한글 커밋 메시지를 작성하여 자동으로 커밋을 수행합니다.

## Context Trigger
- 유저가 "git 작업해줘" 또는 "정리해서 커밋해줘"라고 요청할 때

## Instructions
1. **분석**: `git diff`를 통해 변경된 모든 파일을 확인합니다.
2. **그룹화**: 변경 사항을 기능별/도메인별로 그룹화합니다.
   - 예: UI 수정 사항, API 로직 수정, 설정 파일 변경 등은 각각 별도의 커밋으로 분리.
3. **메시지 작성**: 각 그룹에 대해 아래 규칙으로 한글 메시지를 작성합니다.
   - 형식: `[태그] 요약 설명` (태그 예: feat, fix, refactor, chore 등)
   - 어조: 명확하고 간결한 한국어 구어체/문어체 혼용 가능.
4. **실행**: 
   - 각 그룹에 속한 파일들만 `git add` 합니다.
   - 작성된 메시지로 `git commit`을 실행합니다.
   - 모든 그룹에 대해 반복합니다.

## Constraints
- 한 번에 너무 많은 파일을 하나의 커밋에 넣지 마세요.
- 모호한 변경사항은 유저에게 "이 부분은 어떤 의도인가요?"라고 질문한 뒤 진행하세요.
