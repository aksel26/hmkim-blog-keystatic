## 카테고리

현재 지원되는 카테고리는 다음과 같습니다:

| 카테고리 | 설명 | 예시 주제 |
|---------|------|----------|
| `tech` | 기술 관련 포스트 | React 튜토리얼, TypeScript 팁, 개발 도구 비교 |
| `life` | 일상/라이프스타일 | 여행기, 제품 리뷰, 일상 에세이 |

## 템플릿 유형

각 카테고리에서 사용할 수 있는 템플릿 유형입니다:

### 1. Default (기본)

일반적인 블로그 포스트 형식입니다.

**구조:**
- 서론: 주제 소개 및 독자 관심 유도
- 본론: 핵심 내용 전달 (2-3개 섹션)
- 결론: 요약 및 마무리

**적합한 경우:**
- 일반적인 정보 전달
- 개인 경험 공유
- 단순한 주제 설명

### 2. Tutorial (튜토리얼)

단계별 가이드 형식입니다.

**구조:**
- 개요: 무엇을 배울지, 선행 조건
- 단계별 설명: Step 1, Step 2, ...
- 전체 코드/결과물
- 마무리: 다음 단계 제안

**적합한 경우:**
- How-to 가이드
- 설치/설정 방법
- 프로젝트 구축 과정

### 3. Comparison (비교 분석)

두 가지 이상의 옵션을 비교하는 형식입니다.

**구조:**
- 비교 대상 소개
- 비교 기준 설명
- 상세 비교 (표 형식 권장)
- 결론: 상황별 추천

**적합한 경우:**
- 라이브러리/프레임워크 비교
- 제품 비교
- 방법론 비교

### 4. Deep-dive (심층 분석)

특정 주제를 깊이 있게 다루는 형식입니다.

**구조:**
- 주제 배경
- 핵심 개념 설명
- 내부 동작 원리
- 실제 적용 사례
- 고급 팁

**적합한 경우:**
- 기술 내부 동작 원리
- 복잡한 개념 설명
- 아키텍처 설계

### 5. Tips (팁 모음)

빠른 팁과 트릭을 모아놓은 형식입니다.

**구조:**
- 간단한 소개
- 팁 목록 (번호 또는 제목별)
- 각 팁에 대한 간결한 설명
- 보너스 팁 (선택)

**적합한 경우:**
- 생산성 향상 팁
- 숨겨진 기능 소개
- 베스트 프랙티스 모음

## 워크플로우

블로그 포스트 생성 워크플로우는 다음 순서로 진행됩니다:

```
1. Research (15%)
   └─ 주제에 대한 리서치 수행

2. Write (30%)
   └─ 초안 작성

3. Review (45%)
   └─ AI SEO & 기술 검토

4. Create (60%)
   └─ 콘텐츠 개선 및 메타데이터 생성

5. Validate (75%)
   └─ 콘텐츠 검증

6. Human Review (85%)
   └─ 사용자 검토 (승인/수정 요청/보류)
   │
   ├─ 승인 → Deploy 진행
   ├─ 수정 요청 → Write(2단계)부터 재실행
   └─ 보류 → on_hold 상태로 대기

7. Deploy (95%)
   └─ Git 브랜치 생성 및 PR 생성
```

### Human Review 옵션

| 옵션 | 설명 | 후속 동작 |
|-----|------|----------|
| **승인** | 콘텐츠 승인 | Deploy 단계로 진행 |
| **수정 요청** | 피드백 반영 요청 | Write 단계부터 재실행 |
| **재작성** | 전체 재작성 요청 | Write 단계부터 재실행 |
| **보류** | 나중에 검토 | on_hold 상태로 대기 |

## 메타데이터 구조

생성되는 MDOC 파일의 Frontmatter 구조입니다:

```yaml
---
title: "포스트 제목"           # 필수, 30-60자 권장
summary: "포스트 요약"         # 필수, 50-150자 권장
keywords:                      # 필수, 3개 이상
  - 키워드1
  - 키워드2
  - 키워드3
status: draft                  # draft 또는 published
tags:                          # 필수, 3-5개
  - 태그1
  - 태그2
  - 태그3
createdAt: 2024-01-21         # YYYY-MM-DD 형식
updatedAt: 2024-01-21         # YYYY-MM-DD 형식
---
```

### 필드 설명

| 필드 | 타입 | 필수 | 설명 |
|-----|------|-----|------|
| `title` | string | O | 포스트 제목 (30-60자 권장) |
| `summary` | string | O | 포스트 요약 (50-150자 권장) |
| `keywords` | string[] | O | SEO 키워드 (최소 3개) |
| `status` | string | O | 상태 (draft/published) |
| `tags` | string[] | O | 태그 (3-5개) |
| `createdAt` | string | O | 생성일 (YYYY-MM-DD) |
| `updatedAt` | string | O | 수정일 (YYYY-MM-DD) |

## SEO 가이드라인

검색 엔진 최적화를 위한 가이드라인입니다.

### 제목 (Title)

- **길이**: 30-60자 권장
- **키워드**: 주요 키워드를 앞쪽에 배치
- **형식**: 명확하고 매력적으로

**좋은 예:**
- "React 18 새로운 기능 완벽 가이드"
- "TypeScript 타입 가드 활용법 5가지"

**나쁜 예:**
- "React" (너무 짧음)
- "React 18에서 새롭게 추가된 모든 기능들과 그것들을 활용하는 방법에 대한 완벽한 가이드" (너무 김)

### 요약 (Summary)

- **길이**: 50-150자 권장
- **내용**: 포스트의 핵심 가치를 전달
- **키워드**: 자연스럽게 포함

### 키워드 (Keywords)

- **개수**: 최소 3개, 권장 5-7개
- **선정**: 검색량이 있는 관련 키워드
- **배치**: 제목, 요약, 본문에 자연스럽게 분포

### 헤딩 구조

```
# H1 - 페이지당 하나 (제목)
## H2 - 주요 섹션
### H3 - 하위 섹션
#### H4 - 세부 항목
```

### 콘텐츠 길이

- **최소**: 500자 이상
- **권장**: 1,500-3,000자
- **심층 분석**: 3,000자 이상
