
# agent-web 기능 고도화

### 1단계: 프로젝트 초기화 및 기본 환경 설정

가장 먼저 기술 스택을 정의하고 기본 폴더 구조를 잡는 단계입니다.

> **Claude Code 프롬프트:**
> "Next.js, Tailwind CSS v4, shadcn/ui를 사용하는 블로그 포스팅 자동화 웹 서비스 프로젝트.
> **주요 요구사항:**
> 1. `lucide-react` 아이콘 라이브러리와 `v0` 스타일의 shadcn UI 컴포넌트들을 기본적으로 설치해줘.
> 2. 다크 모드를 지원하는 레이아웃을 구성해줘.
> 3. `next-themes`를 설치하고 설정해줘."
> 
> 

---

### 2단계: 카테고리별 동적 입력 폼 구현

주식, 요리, 영화 등 카테고리에 따라 입력 필드가 변하는 핵심 UI를 만드는 단계입니다.

> **Claude Code 프롬프트:**
> "카테고리에 따라 동적으로 변하는 '블로그 인풋 폼' 컴포넌트를 작성해줘.
> **포함할 기능:**
> 1. `React Hook Form`과 `Zod`를 사용하여 유효성 검사를 수행할 것.
> 2. **카테고리 선택(Select):** 주식, 요리, 영화, 맛집, IT 중 하나를 선택.
> 3. **조건부 렌더링:** >    - **주식:** 종목명, 분석 종류(차트/실적), 투자 관점 필드.
> * **요리:** 메뉴명, 난이도, 핵심 재료, 꿀팁 필드.
> * **영화:** 작품명, 별점(Slider), 명대사 필드.
> * **맛집:** 가게명, 위치, 주문 메뉴, 재방문 의사 필드.
> 
> 
> 4. 모든 카테고리 공통으로 '말투(Tone)', '타겟 독자', '핵심 키워드' 필드를 포함해줘."
> 
> 

---

### 3단계: 멀티 에이전트 활용

입력된 데이터를 프롬프트로 조합하여 AI로부터 본문을 받아오는 기능을 구현합니다.

현재 에이전트들의 역할에 따라서 각각 에이전트가 진행하고 있는 상황에 대해서 로그를 나타냅니다.

에이전트가 초안을 작성할 때에는 Streaming방식으로 보여줍니다.


---

### 4단계: 에디터 및 배포 자동화 (Keystatic/Notion 연동)

생성된 글을 검토하고 실제 블로그로 전송하는 단계입니다.

> **Claude Code 프롬프트:**
> "생성된 마크다운 본문을 편집할 수 있는 에디터 페이지를 만들어줘.
> **기능 요구사항:**
> 1. 현재 mdoc 파일을 생성하고, PR 요청을 하는 로직을 아래처럼 수정해줘.
> 2. `react-markdown`을 사용하여 미리보기를 지원하거나, `TipTap` 에디터를 연동해줘.
> 3. **배포 버튼:** 클릭 시 현재 작성된 데이터를 Keystatic CMS 형식에 맞춰 GitHub API를 통해 Content 저장소로 Push하는 스켈레톤 코드를 작성해줘.
> 4. SEO 최적화 체크리스트(제목 길이, 키워드 포함 여부 등)를 사이드바에 표시해줘."
> 
> 

---

### 5단계: (고급) RSS 수집 및 예약 자동화 (나중에 구현)

자동화의 정점인 수집 및 스케줄링 단계입니다.

> **Claude Code 프롬프트:**
> "특정 뉴스 사이트의 RSS 피드를 수집하여 주식 카테고리의 인풋 데이터를 자동으로 채워주는 Server Action을 작성해줘.
> 1. `rss-parser`를 사용하여 최근 뉴스를 가져올 것.
> 2. 수집된 뉴스를 AI 요약 노드에 전달하여 자동으로 폼 필드를 채우는 기능을 제안해줘.
> 3. Upstash QStash나 Vercel Cron Jobs를 사용하여 매일 특정 시간에 자동으로 포스팅을 생성하고 임시 저장하는 구조를 설계해줘."
